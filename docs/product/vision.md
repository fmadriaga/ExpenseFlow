# Vision

## Nombre tentativo
ExpenseFlow

## Problema
Hoy el registro de tickets y facturas se hace de forma manual o semimanual,
lo que dificulta mantener una trazabilidad consistente de los gastos,
obtener datos históricos y convertir esa información en presupuesto,
planificación y análisis familiar.

## Visión del producto
ExpenseFlow será una solución personal/familiar para capturar tickets de compra,
extraer datos estructurados con OCR y almacenarlos de forma confiable para su
posterior análisis en una app de control de gastos, presupuesto y planificación.

## Objetivo del MVP actual
Resolver de punta a punta el flujo de:
1. detectar tickets nuevos en una carpeta sincronizada
2. leerlos mediante OCR
3. normalizar sus datos
4. almacenarlos en SQLite
5. mover el archivo a processed o error

## Usuarios objetivo iniciales
- Usuario principal: Fernando
- Uso secundario potencial: familiares cercanos

## Beneficios esperados
- Menor carga manual
- Histórico consistente de gastos
- Base de datos lista para futuras funcionalidades
- Proyecto reusable y demostrable en portfolio

## Fuera de alcance del MVP
- Presupuestos
- Objetivos financieros
- División de gastos entre familiares
- App móvil
- Multiusuario
- Integración bancaria
